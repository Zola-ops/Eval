# ScoreEval - AI 评估器项目

基于大语言模型的智能评估系统，支持三种API调用渠道方案。

## 📦 三种评估方案

| 方案 | 脚本文件 | API | 适用场景 |
|------|---------|-----|----------|
| **智谱API渠道** | `scoreeval_zhipu.py` | 智谱官方API (glm-4.5) | 推荐，稳定高效 |
| **Coze API渠道** | `scoreeval_coze_api.py` | Coze HTTP API | 需要Coze平台 |
| **Coze SDK渠道** | `scoreeval_coze_sdk.py` | Coze SDK | 需要Coze平台 |

## 🚀 快速开始

### 方案一：智谱API方案（推荐）

```bash
# 单条评估
python3 standalone_evaluator_zhipu.py --query "什么是人工智能?" --answer "人工智能是..."

# 批量评估
python3 standalone_evaluator_zhipu.py --excel input.xlsx --output result.xlsx
```

**配置文件**：`evaluator_config.json`

```json
{
  "config": {
    "model": "glm-4.5",
    "temperature": 0.01,
    "top_p": 0.7,
    "max_completion_tokens": 29000,
    "thinking": "disabled"
  },
  "api_url": "https://open.bigmodel.cn/api/paas/v4/chat/completions",
  "api_token": "your-api-key"
}
```

### 方案二：Coze API方案

```bash
# 单条评估
python3 scoreeval_coze_api.py --query "问题" --answer "回答"

# 批量评估
python3 scoreeval_coze_api.py --excel input.xlsx --output result.xlsx
```

**配置文件**：`evaluator_config.json`

需要设置Coze API相关配置。

### 方案三：Coze SDK方案

```bash
# 单条评估
python3 standalone_evaluator.py --query "问题" --answer "回答"

# 批量评估
python3 standalone_evaluator.py --excel input.xlsx --output result.xlsx
```

需要安装Coze SDK：
```bash
pip install coze-api
```

## ✨ 核心功能

### 1. 智能考点生成
- 深入分析问题意图
- 提取核心需求和关键要素
- 生成可判定的考点列表（3-6个）

### 2. 准确评分
- 基于考点的客观判定
- 二分类评分（0/1）
- 详细的评分理由和依据

### 3. 上下文感知
- 支持多轮对话评估
- 自动维护会话历史
- 考虑上下文关联性

### 4. 批量评估
- Excel批量处理
- 实时进度显示
- 错误处理机制

## 📊 数据格式

### 单条评估
- `--query`: 用户问题
- `--answer`: 模型回答
- 可选：`--date`, `--demand`

### 批量评估
Excel文件必须包含以下列：

| 列名 | 必填 | 说明 |
|------|------|------|
| query | ✅ | 用户问题 |
| answer | ✅ | 模型回答 |
| session_id | ❌ | 会话ID（用于上下文感知） |
| query_id | ❌ | 问题ID |
| date | ❌ | 问题日期 |
| demand | ❌ | 需求描述 |

## 📤 输出格式

### 单条评估输出

```json
{
  "keypoint_generation": {
    "thinking": "分析思考过程",
    "analysis": "问题分析",
    "main_demand": "核心需求",
    "key_point": ["考点1", "考点2", "考点3"]
  },
  "scoring": {
    "thinking": "评估思考过程",
    "analysis": "回答分析",
    "key_point_result": [
      {"point": "考点1", "satisfied": true, "evidence": "依据"},
      {"point": "考点2", "satisfied": false, "evidence": "依据"}
    ],
    "score": 0,
    "reason": "评分理由"
  }
}
```

### 批量评估输出

Excel文件新增列：
- `keypoint_generation`: 考点生成结果（JSON字符串）
- `scoring`: 评分结果（JSON字符串）
- `evaluated_at`: 评估时间
- `error`: 错误信息（如有）

## 🔧 配置说明

### 模型参数

在 `evaluator_config.json` 中配置：

```json
{
  "config": {
    "model": "glm-4.5",
    "temperature": 0.01,
    "top_p": 0.7,
    "max_completion_tokens": 29000
  }
}
```

### Prompt模板

可自定义四个Prompt：
- `sp`: 考点生成的System Prompt
- `up`: 考点生成的User Prompt模板
- `scorer_sp`: 评分的System Prompt
- `scorer_up`: 评分的User Prompt模板

## 🐛 常见问题

### Q: API调用失败怎么办？

检查以下几点：
1. API Key是否正确
2. 网络连接是否正常
3. API端点是否可用
4. 账户余额是否充足

### Q: Excel文件格式错误？

确保Excel包含必要列：
- 必须列：`query`, `answer`

### Q: JSON解析错误？

可能原因：
- 模型输出包含未转义的换行符
- max_completion_tokens不足导致截断

解决方案：
- 增加 `max_completion_tokens` 值
- 降低 `temperature` 使输出更稳定
